# PaperCraft AI

一个面向 LaTeX 论文的 AI 写作增强编辑器 MVP。选中文本后，应用会携带前后各 3 个段落生成保守、学术强化和精简三个版本，并解释每个版本的修改理由；用户可一键替换及撤销，也可以只查看建议而不改动原文。

## 启动

macOS 可以直接双击 `启动论文编辑器.command`，启动器会自动打开浏览器。关闭启动器的终端窗口即可停止服务。

也可以在终端手动启动：

```bash
python3 server.py
```

打开 `http://localhost:8000`。未设置密钥时使用离线演示结果。

## 打开论文

点击顶部“打开项目”可以选择整个论文文件夹；点击“打开文件/ZIP”可以直接导入压缩包或单个文件。长期编辑 ZIP 项目时，推荐点击“从 ZIP 创建项目”，应用会在本应用目录下的 `projects/` 中创建 ZIP 同名项目文件夹并把内容写入真实目录。导入后左侧会显示项目文件树：

- `.tex`、`.bib`、`.sty`、`.rtx` 等 LaTeX 文本文件可打开和编辑
- PDF 与常见图片可以直接预览
- `main.tex` 会被优先识别为主文件

项目内容仅在本机浏览器和本地服务中处理。完整项目会自动保存在浏览器本地数据库中，下次启动时自动恢复。

- 使用“打开项目”并授予目录写权限时，文本修改会自动写回原文件。
- 使用“从 ZIP 创建项目”时，应用会在 `projects/` 下创建 ZIP 同名子目录；若已存在同名目录，会自动创建 `-2`、`-3` 等编号目录，不覆盖已有项目。
- 使用普通 ZIP、拖拽或普通文件选择器导入时，修改会自动保存到浏览器项目缓存，但不会覆盖原始 ZIP 或文件。
- 顶部状态会显示“保存中”“已保存到原文件”“已自动保存”或“保存失败”。

界面中的文件树、编辑区、AI 控制区和结果区可以通过分隔条左右拖动调整宽度。LaTeX 长行会按照编辑区当前宽度自动换行，布局宽度会在下次启动时恢复。

写作目标通过下拉列表选择，四个目标各自带有可编辑的默认提示词。提示词位于“上下文已启用”说明下方，修改后自动保存在浏览器中，润色时以当前编辑内容作为最终写作要求；“恢复默认”只重置当前目标的提示词。

AI 控制区提供“写作反馈”卡片，点击“分析当前文件”会生成 3-5 条中文非侵入式反馈。反馈只提示表达、逻辑、过渡或证据问题，不会自动修改源码。

右侧结果区提供“对话”标签，可以直接围绕整个项目向大模型提问。项目级对话会尽量携带所有文本文件上下文，二进制资源只作为清单提供；模型生成的跨文件修改建议必须先展示预览，只有点击“应用这处修改”后才会写入对应文本文件并进入自动保存。

## 论文工作台

- AI 结果卡片显示修改理由和逐词 Diff，绿色表示新增、红色表示删除。
- 右侧“对话”标签支持项目级问答和需确认后应用的精确文本替换建议。
- 顶部“导出 ZIP”会导出当前编辑后的完整项目及二进制资源。
- 在 `\cite{}`、`\ref{}`、`\input{}` 或 `\includegraphics{}` 上双击或按住 `Command/Ctrl` 点击可以跳转。
- 使用 `Command/Ctrl+F` 打开搜索替换。
- 点击“编译 PDF”会在本机临时目录编译，成功后直接切换到右侧 PDF 标签预览；预览区提供“下载 PDF”和“全屏浏览”按钮，全屏浏览会在新标签打开浏览器独立 PDF 预览器并请求 250% 缩放。错误会弹出诊断面板并可点击跳回源码。“保存后自动编译”开关常显在编译按钮上方。

本地编译需要安装 `latexmk`、`pdflatex` 或 `xelatex`。当前开发机已安装 BasicTeX 2026，并补齐示例论文所需的 REVTeX 4.1、CJK、newtx、overpic 等宏包。服务会检查 `/Library/TeX/texbin` 等常见目录，也可通过 `LATEX_ENGINE` 指定完整路径。没有工具链时应用会显示明确提示。直接使用 XeLaTeX/PDFLaTeX 时，服务会在需要时执行 BibTeX 并再排版两次；整个构建禁用 shell-escape，总超时为 90 秒。当前尚未实现基于 SyncTeX 的源码与 PDF 双向定位，也尚未接入 Monaco 语法高亮。

编译结果优先通过本地 `/api/pdf/` 地址预览，并支持浏览器分段读取；前端会先切到可见 PDF 标签再加载预览，并验证该地址。浏览器 PDF 查看器加载异常只显示中文预览提示，不会覆盖编译成功状态或弹出底层对象错误。通过“打开项目”导入的目录资源会在编译或导出前尝试从文件句柄重新读取；保存项目状态时会把 PDF/图片真实内容写入 IndexedDB 缓存，浏览器重启后可直接继续编译。双击启动脚本会检查服务版本，并自动重启占用 8000 端口的旧版 PaperCraft 服务。

## 接入模型

模型统一在 `model_config.json` 中配置。修改配置后无需重启服务，下一次润色请求会自动读取最新内容。

1. 将 `active_provider` 改为要使用的配置名称，例如 `deepseek`、`anthropic` 或 `relay`。
2. 在对应提供商中填写 `api_key`。
3. 根据账号实际可用模型修改 `model`。

中转站示例：

```json
{
  "active_provider": "relay",
  "providers": {
    "relay": {
      "type": "openai_compatible",
      "api_key": "sk-your-key",
      "base_url": "https://your-relay.example.com/v1",
      "model": "your-model-name",
      "json_mode": true,
      "extra_headers": {}
    }
  }
}
```

内置配置包括：

- OpenAI 及 OpenAI 兼容中转站
- Anthropic Claude 原生接口
- Google Gemini 原生接口
- DeepSeek
- 阿里云通义千问
- 月之暗面 Kimi
- 智谱 GLM
- 硅基流动

`base_url` 可以填写服务根地址或完整的 `chat/completions` 地址。少数中转站不支持 JSON Mode 时，将 `json_mode` 设为 `false`；需要特殊请求头时填写 `extra_headers`。

密钥只由本地 Python 服务读取，不会发送到浏览器。也可以把 `api_key` 写成 `env:MY_MODEL_KEY`，再通过系统环境变量提供密钥。`model_config.json` 已加入 `.gitignore`，不要将真实密钥提交到版本库。

模型请求优先使用系统 `curl`，以兼容 macOS 自带 Python 的旧版 LibreSSL。API Key 通过仅当前用户可读的临时请求头文件传递，并在请求结束后立即删除；其他系统没有 `curl` 时回退到 Python urllib。
