# PaperCraft AI 下一阶段修改计划

> 本文件只记录尚未完成、部分完成但需要升级、或明确暂缓的事项。已完成能力以“当前基线”简要说明，不重复列入待办。  
> 后续每次实现代码、UI、API、配置或用户可见行为时，仍必须同步更新 `latex_ai_editor_prd.md`。

---

## 0. 当前基线

PaperCraft 当前已经具备以下基础能力：

- 本地 Web App、macOS 双击启动、本地模型配置与离线演示模式。
- 完整 LaTeX 项目导入、文件树、文本编辑、资源预览、自动保存和项目恢复。
- 选区 AI 润色：A/B/C 多版本、修改理由、一键替换、逐词 Diff。
- 手动写作反馈：点击“分析当前文件”后返回 3-5 条非侵入式中文建议。
- 项目级 AI 对话：可携带项目上下文问答，并生成需确认后应用的精确文本替换建议。
- 当前文件搜索替换、引用/标签/输入文件/图片资源跳转。
- 本地 LaTeX 编译、错误诊断源码定位、PDF 实时预览、PDF 下载和独立 PDF 预览器打开。
- PDF/图片等资源持久化缓存，`.rtx` 等 LaTeX 辅助文本文件已按文本处理。

## P0. 已知维护事项

### P0.1 编译与资源缓存回归观察

**目标**  
持续确认重启后项目恢复、二进制资源缓存、`.rtx` 文本辅助文件和 PDF 预览地址不会再次触发对象失效问题。

**当前状态**  
已修复 PDF/图片旧 File 快照失效、`.rtx` 误判为未知资源、PDF token 失效回退等问题；仍需要在后续功能修改中保留回归验证。

**主要模块**  
前端项目缓存、文件类型识别、`/api/project`、`/api/compile`、`/api/pdf/`。

**验收标准**  
重启编辑器后无需重新打开项目即可编译示例论文；ZIP 导入项目可编译；PDF 预览、下载和独立预览器打开正常。

**风险/依赖**  
浏览器文件句柄权限、IndexedDB 存储容量、Safari/Chrome PDF 查看器行为差异。

## P1. 编辑器基础升级

### P1.1 接入 Monaco Editor

**目标**  
用 Monaco Editor 替换当前轻量 textarea，获得更稳定的编辑体验、滚动性能、快捷键和后续语言能力扩展基础。

**当前状态**  
当前使用轻量文本编辑器，行号、软换行、选区和滚动位置由前端自行维护。

**主要模块**  
编辑器渲染层、选区锁定、自动保存、查找替换、跳转定位、布局恢复。

**验收标准**  
打开大型 `.tex` 文件不卡顿；现有选区润色、一键替换、撤销、自动保存、文件切换、滚动恢复和快捷键行为保持可用。

**风险/依赖**  
Monaco 体积较大；需要重新适配选区锁定、Diff 应用和行号/滚动状态保存。

### P1.2 LaTeX 语法高亮

**目标**  
在编辑器中显示 LaTeX 命令、环境、数学公式、引用、注释等基础语法高亮。

**当前状态**  
未实现语法高亮；PRD 将其列为后续升级项。

**主要模块**  
Monaco language configuration、tokenizer、主题样式。

**验收标准**  
常见命令如 `\section{}`、`\cite{}`、`\ref{}`、数学环境、注释能够以不同颜色显示；高亮不改变源文本。

**风险/依赖**  
依赖 Monaco 接入；复杂 LaTeX 宏不要求完整解析，优先覆盖常见论文写作场景。

### P1.3 大文件增量渲染

**目标**  
降低长篇论文、长行和大量文件场景下的编辑器卡顿。

**当前状态**  
当前行号和软换行仍偏全量计算，PRD 标记为未开始。

**主要模块**  
编辑器渲染层、文件切换、滚动恢复、项目缓存。

**验收标准**  
长篇 `.tex` 文件打开、滚动、输入、切换文件时保持流畅；不会因为行号重算阻塞界面。

**风险/依赖**  
依赖 Monaco 或等价编辑器虚拟化能力；需要避免破坏现有自动保存节奏。

## P2. 项目级编辑能力

### P2.1 跨文件全项目搜索

**目标**  
支持在整个论文项目中搜索关键词、LaTeX 命令、引用 key、标签和术语。

**当前状态**  
当前仅支持当前文件搜索替换。

**主要模块**  
项目文件索引、搜索面板、结果列表、文件打开与定位。

**验收标准**  
能够搜索所有文本文件；结果显示文件路径、行号和上下文；点击结果跳转到对应文件和位置。

**风险/依赖**  
大项目搜索性能；需要排除二进制文件和生成文件；搜索结果要与自动保存后的最新内容一致。

### P2.2 跨文件替换与重命名

**目标**  
支持对项目内术语、标签、引用 key 或变量名进行可预览的跨文件替换。

**当前状态**  
当前只有当前文件替换；项目级 AI 对话可以生成精确文本替换建议，但不是通用搜索替换工具。

**主要模块**  
跨文件搜索结果、替换预览、确认应用、自动保存、撤销策略。

**验收标准**  
替换前展示影响文件和匹配数量；用户确认后才写入；写入后进入现有自动保存流程；不处理二进制文件。

**风险/依赖**  
误替换风险高；需要提供预览、逐项选择或撤销路径。

### P2.3 编译配置

**目标**  
允许用户配置主文件、编译引擎、BibTeX/Biber、构建参数和项目级默认编译选项。

**当前状态**  
当前自动优先找 `main.tex`，自动检测 `latexmk`、`xelatex`、`pdflatex`，直接引擎时自动 BibTeX 和两轮重排。

**主要模块**  
项目配置存储、编译 UI、`/api/compile` 请求字段、后端命令构造。

**验收标准**  
用户可选择非 `main.tex` 主文件；可选择 XeLaTeX/PDFLaTeX/latexmk；配置随项目保存；默认仍保持自动检测。

**风险/依赖**  
需要继续禁用危险选项，如 shell-escape；配置错误时必须给出清晰诊断。

## P3. PDF 与源码联动

### P3.1 SyncTeX 正向定位

**目标**  
从源码行跳转到 PDF 对应页面和位置。

**当前状态**  
编译响应已保留 SyncTeX 产物，但前端尚未解析或使用。

**主要模块**  
SyncTeX 解析、源码位置映射、PDF 预览控制、跳转 UI。

**验收标准**  
用户在源码中触发定位后，PDF 跳到对应页附近；无法定位时给出中文提示，不影响编译结果。

**风险/依赖**  
浏览器内置 PDF 查看器可控性有限，可能需要 PDF.js。

### P3.2 PDF 反向定位源码

**目标**  
用户在 PDF 中点击或选择位置后，定位到对应 `.tex` 源码行。

**当前状态**  
未开始；当前 PDF 预览主要依赖浏览器内置 PDF 查看器。

**主要模块**  
PDF 点击坐标捕获、页面坐标映射、SyncTeX 反向查询、源码打开与滚动。

**验收标准**  
点击 PDF 文本附近后，左侧打开对应源文件并选中或滚动到相关行。

**风险/依赖**  
强依赖自定义 PDF 渲染层；若继续使用浏览器内置 PDF 查看器，可能无法捕获页面坐标。

## P4. AI 写作智能升级

### P4.1 实时被动反馈

**目标**  
把当前手动“分析当前文件”升级为可控的后台反馈机制，提示弱论断、过渡不清、结构问题和证据不足。

**当前状态**  
已有手动写作反馈；不会实时扫描，也没有行内提示。

**主要模块**  
反馈触发策略、节流/防抖、反馈结果缓存、UI 展示、模型调用限流。

**验收标准**  
用户编辑后在合理延迟内看到非侵入式反馈；不会自动改文；可关闭实时反馈；不会频繁打断写作。

**风险/依赖**  
模型费用和延迟；实时调用可能干扰写作，需要强节流和手动开关。

### P4.2 行内高亮与 Hover Suggestion

**目标**  
在编辑器中以轻量方式标记问题句，并在 hover 或点击时显示建议。

**当前状态**  
无行内高亮；反馈只在控制区列表展示。

**主要模块**  
编辑器 decoration、反馈位置映射、hover 卡片、建议交互。

**验收标准**  
弱句、过渡、证据、结构类问题可在文本旁边标记；hover 显示中文说明和建议；不修改源码。

**风险/依赖**  
依赖 Monaco decoration；需要稳定的文本位置映射，避免编辑后标记错位。

### P4.3 结构建议面板

**目标**  
提供文档结构级建议，如段落拆分、段落移动、补充过渡句、缺少逻辑块提示。

**当前状态**  
项目级 AI 对话可以讨论结构，但没有专门结构分析面板，也不会生成结构化建议卡片。

**主要模块**  
项目上下文收集、结构分析 prompt、建议卡片 UI、跳转到相关段落。

**验收标准**  
分析后返回若干结构建议；每条建议包含位置、问题、建议操作和理由；用户确认前不改源码。

**风险/依赖**  
长文上下文限制；结构建议容易过度主观，需要强调“建议而非自动改写”。

## P5. 术语与安全改写

### P5.1 术语表

**目标**  
允许用户为项目维护术语表，确保专业名词、缩写、变量名和固定译法保持一致。

**当前状态**  
系统 prompt 通用保护 LaTeX 命令、公式、引用和术语，但没有用户可配置术语表。

**主要模块**  
术语表 UI、项目配置存储、润色/反馈/对话 prompt 注入。

**验收标准**  
用户可新增、编辑、删除术语；AI 润色时保持术语不被随意替换；术语表随项目恢复。

**风险/依赖**  
术语过多会占用模型上下文；需要限制长度并优先保留当前文件相关术语。

### P5.2 禁止修改列表与统一写法

**目标**  
允许用户指定绝不能改写的词、短语、符号、命令，或指定统一写法。

**当前状态**  
无项目级禁止修改列表；当前只靠系统 prompt 的通用保护。

**主要模块**  
项目配置、AI 请求构造、替换前校验、提示词约束。

**验收标准**  
AI 返回内容不得改变禁止项；替换前可检测明显违反规则的结果并提醒用户。

**风险/依赖**  
模型可能仍违反约束；需要前端或后端增加替换前安全检查。

### P5.3 Citation-aware Suggestions

**目标**  
让 AI 在写作反馈中识别关键背景、方法、对比结论是否可能需要引用支持。

**当前状态**  
离线演示反馈中有简单“较长论述没有引用”的提示；未做真正 citation-aware 分析。

**主要模块**  
反馈 prompt、BibTeX key 清单、引用上下文分析、建议展示。

**验收标准**  
反馈能指出可能缺少引用的位置，并可建议检查相关 BibTeX 条目；不自动生成虚假引用。

**风险/依赖**  
不能编造 Citation；必须保持建议性质，让用户确认文献来源。

## 执行原则

- AI 永远是建议系统，不默认覆盖用户文本。
- 所有自动生成的修改必须先展示给用户确认。
- LaTeX 命令、公式、引用、标签、路径和术语必须优先保护。
- 每次实现代码、UI、API、配置或用户可见行为时，同步更新 `latex_ai_editor_prd.md`。
- 每个任务完成前必须跑最相关的验证，并确认实现与 PRD 描述一致。
