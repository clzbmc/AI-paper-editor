# AI LaTeX Paper Editor 产品需求文档

> 产品代号：PaperCraft AI  
> 版本：v0.8 CodeMirror Workbench  
> 更新日期：2026-06-25  
> 当前形态：本地 Web App

---

## 1. 项目目标

构建一个面向科研论文 LaTeX 项目的 AI 写作增强工具。

用户可以导入完整论文项目，在 LaTeX 文本中选中任意内容，结合论文上下文调用大模型，获得多个带修改理由的学术润色版本，并由用户决定是否一键替换原文本。同时，系统可以对当前文本给出非侵入式写作反馈，且必须保持公式、命令、引用和专业术语不被破坏。

产品定位：**Cursor for Papers（论文版智能编辑器）**。

---

## 2. 核心用户

- PhD 学生
- 科研人员
- CFD、DRL、AI、机械、流体等工程科研用户
- 英文学术论文写作者

---

## 3. 核心价值

### 3.1 解决的问题

- LaTeX 局部修改效率低
- Grammarly 等工具不能理解 LaTeX 结构
- 使用通用聊天工具需要反复复制粘贴
- AI 改写容易破坏公式、引用和命令
- 论文项目包含多个 TeX、BibTeX 和图片文件，单文件编辑不足以理解项目结构

### 3.2 产品能力

- 选中即润色
- 自动携带上下文
- 保留 LaTeX 结构
- 提供多个改写版本
- 解释每个改写版本的修改理由
- 提供不自动改文的轻量写作反馈
- 一键替换并支持撤销
- 导入并浏览完整论文项目
- 查看参考文献文件和论文资源

---

## 4. MVP 用户流程

1. 用户双击 `启动论文编辑器.command`。
2. 本地服务启动并自动打开浏览器。
3. 系统自动恢复上次打开的完整论文项目；用户也可以选择新的论文文件夹、从 ZIP 创建项目，或打开单个可编辑文件。
4. 系统建立项目文件树并优先打开 `main.tex`。
5. 用户在编辑器中选中需要润色的内容。
6. 系统读取选区前后各 3 个段落作为上下文。
7. AI 返回保守修订、学术强化、精简表达三个版本，并解释每个版本的修改理由。
8. 用户复制结果或一键替换原选区。
9. 用户也可以输入中文草稿或写作意图，一键生成贴合论文上下文和当前写作目标的英文内容，并在确认后插入光标处。
10. 用户也可以点击“分析当前文件”查看 10 条写作反馈，反馈不自动修改文本。
11. 文本修改自动保存到浏览器项目缓存；拥有目录写权限时同时写回原文件。

## 5. 功能需求

### 5.1 本地启动

- 提供 macOS 双击启动文件
- 自动启动 Python 本地服务
- 自动在默认浏览器打开应用
- 重复启动时直接打开已运行的应用
- 关闭终端窗口或按 `Ctrl+C` 停止服务

### 5.2 项目导入

支持以下导入方式：

- 选择完整论文文件夹
- 从 ZIP 创建可见项目文件夹
- 导入单个可编辑文本文件或 `.docx` 文档
- 将单个可编辑文件拖入编辑区

入口职责：

- “打开文件”只用于打开单个可编辑文件，不再直接导入 ZIP
- ZIP 项目必须通过“从 ZIP 创建项目”导入并创建可见项目文件夹
- 选择或拖入 ZIP 时提示使用“从 ZIP 创建项目”
- `.docx` 仅提取正文为纯文本进行编辑，不保留 Word 样式、图片、批注、修订痕迹或版式
- 旧版 `.doc` 暂不支持，用户需另存为 `.docx` 或文本格式

ZIP 安全限制：

- 压缩包最大 50 MB
- 解压后总内容最大 100 MB
- 最多读取 500 个文件
- “从 ZIP 创建项目”由本地服务在应用目录 `projects/` 下创建 ZIP 同名子目录；重名时创建 `-2`、`-3` 等编号目录，不覆盖已有目录

### 5.3 项目文件树

- 显示项目中的相对路径和文件名
- 优先识别路径中的 `main.tex` 为主文件
- 没有 `main.tex` 时打开第一个 `.tex` 文件
- 支持在多个项目文件之间切换
- 切换文件前保留当前浏览器会话中的编辑内容

### 5.4 项目持久化与自动保存

- 使用 IndexedDB 保存完整项目状态，而不是只保存当前文本
- 保存项目名称、当前打开文件、全部可编辑文本和可预览资源
- 应用启动后自动恢复上次打开的项目和文件位置
- 按文件保存并恢复滚动位置、横向位置、光标和文本选区
- 首次打开没有历史状态的文本文件时必须定位到文件开头
- 文本发生修改后 600 ms 自动保存，不要求用户手动点击保存
- 顶部状态明确显示编辑中、保存中、保存完成或保存失败
- 通过“打开项目”取得目录读写权限时，自动写回原文本文件
- 通过“从 ZIP 创建项目”生成的项目，解压出的文本修改必须经本地服务自动写回 `projects/` 下的新建项目文件夹
- 单文件拖拽和普通文件选择器导入时，自动保存到浏览器本地项目缓存
- 浏览器不能取得写权限时，不得声称内容已写入原文件
- 图片和 PDF 资源保存在项目缓存中，以便恢复后继续预览

### 5.5 文件类型

可编辑文本：

- `.tex`、`.latex`
- `.bib`
- `.sty`、`.cls`、`.bst`、`.rtx`
- `.txt`、`.text`、`.md`
- `.json`，用于读取项目侧边配置和 `.papercraft/project_memory.json`
- `.docx`，仅提取正文纯文本，不保留 Word 格式

暂不支持：

- 旧版 `.doc` 二进制 Word 文件

可预览资源：

- `.pdf`
- `.png`、`.jpg`、`.jpeg`
- `.gif`、`.svg`、`.webp`

其他文件显示在文件树中，但当前不提供内容预览。

### 5.6 LaTeX 编辑器

- 支持纯文本 LaTeX 编辑
- 支持文本选区
- 显示行号、光标位置和词数
- 支持 CodeMirror undo/redo 历史
- 自动保存当前项目与全部文本修改
- 支持在项目文本文件之间切换
- 长文本必须在编辑区域内部独立滚动，不能撑开页面布局
- 鼠标位于正文或行号栏时均可使用滚轮上下移动
- 超出编辑区宽度的 LaTeX 长行必须自动软换行，不要求横向滚动
- 自动换行只改变视觉布局，不得在源文件中插入真实换行符
- 编辑区宽度变化时，软换行必须实时重新排版
- 逻辑行换成多行显示后，行号必须保持纵向对齐

> 当前实现为 CodeMirror 6 编辑器，包含行号、软换行、基础搜索替换、LaTeX/sTeX 高亮和大文件增量渲染能力。Monaco Editor 暂不作为当前阶段升级方向。

### 5.7 AI 润色

请求输入：

```json
{
  "text": "selected text",
  "context_before": "previous three paragraphs",
  "context_after": "next three paragraphs",
  "mode": "all | safe | academic | concise",
  "custom_prompt": "user-edited writing instruction"
}
```

返回结果：

```json
{
  "A": "conservative correction",
  "B": "strong academic rewrite",
  "C": "concise rewrite",
  "reasons": {
    "A": "short explanation",
    "B": "short explanation",
    "C": "short explanation"
  }
}
```

版本定义：

- A：保守修订，重点修复语法和清晰度
- B：学术强化，使用更严谨、正式的表达
- C：精简表达，减少冗余并保留原意

写作目标共四项：

- 全部版本：显示 A、B、C 三个结果
- 保守修订：仅显示 A 版本
- 学术强化：仅显示 B 版本
- 精简表达：仅显示 C 版本
- 每个返回版本可以包含一句修改理由，前端在结果卡片中显示
- 旧模型或旧接口未返回 reasons 时，前端仍必须正常显示 A/B/C 与 Diff

### 5.7.1 轻量写作反馈

- 后端提供 POST /api/feedback
- 请求字段包括当前文件路径、当前文件内容、可选选中文本、选区起止位置
- 返回字段为 feedback 数组，每条包含 type、severity、text、suggestion、anchor_text、start、end 和 line
- 写作反馈由用户手动点击“分析当前文件”触发，不做实时自动扫描
- 写作反馈的 text 和 suggestion 必须使用简体中文返回
- 每次返回 10 条反馈建议，前端最多展示 10 条
- 每条反馈必须提供可展开/收起的 suggestion 解决方案或推荐文字
- 每条反馈必须提供跳转入口，点击后左侧编辑器滚动并选中对应原文片段
- 反馈列表区域必须支持纵向滚动，单条反馈可通过叉号删除，删除时播放平滑退场动画且后续条目自动上移
- 反馈只提示表达、逻辑、过渡、结构或证据问题，不自动修改源码
- 当前 v1 只支持反馈跳转定位，不做反馈行内高亮或自动替换
- 当前文件过长时，后端优先截取当前光标或选区附近内容用于分析
- 未配置模型密钥时返回离线演示反馈

### 5.7.2 中文草稿生成英文内容

- 后端提供 POST /api/draft
- 请求字段包括中文草稿 draft、当前文件路径、当前文件内容、光标位置、写作目标和当前目标自定义提示词
- 后端必须截取光标附近论文上下文，避免直接发送超长全文
- 非“全部版本”目标返回字段包括 text、reason 和 demo，可选 provider、model
- “全部版本”目标返回 variants.A/B/C 与 reasons.A/B/C，分别对应保守修订、学术强化和精简表达
- 生成的英文内容必须适合插入论文，不应保留中文草稿原文
- 前端入口位于右侧 AI 工作区的“中文生成”独立标签
- 生成结果以可编辑文本框展示，用户可以手动修改后再复制或插入
- 生成结果不自动写入源码，用户点击对应候选的“插入光标处”后才修改编辑区
- 插入操作应进入浏览器撤销记录，并触发自动保存、行号与字数刷新
- 生成时必须复用当前四个写作目标和当前目标的自定义提示词
- 未配置模型密钥时返回离线演示英文内容

### 5.7.3 项目级 AI 对话

- 右侧结果区提供“对话”标签，用于围绕整个论文项目向大模型提问
- 后端提供 POST /api/chat
- 请求包含对话历史、当前文件路径、尽量完整的项目文本文件上下文和资源清单
- 默认尽量发送全项目文本；超过请求上限时优先完整保留当前文件、主 TeX、BibTeX 和 TeX 文件，其余文本截断并标记
- PDF、图片和未知二进制资源只作为路径、类型和大小清单提供，不发送内容
- 响应包含中文 reply，并可选返回 changes 数组
- changes 使用精确文本替换格式：path、find、replace、reason
- 模型不得直接改文件；所有 changes 必须先展示给用户确认
- 用户点击“应用这处修改”前，前端必须确认目标文本文件仍包含 find 原文
- 应用成功后更新项目文本、当前编辑器内容，并进入现有自动保存流程
- 当前 v1 不支持二进制文件编辑，不支持无确认自动写入

### 5.7.4 项目记忆索引

- 项目加载后后台自动生成项目记忆索引，保存为真实项目目录中的 `.papercraft/project_memory.json`
- 项目加载时必须先扫描并读取已有 `.papercraft/project_memory.json`；若存在可解析记忆，则先展示已有记忆，不自动覆盖重建
- 已有记忆与当前项目指纹不一致时，状态显示内容已变化并建议刷新，保留现有分类供用户查看，只有用户点击刷新记忆才强制重建
- 普通文件夹项目必须通过浏览器文件系统权限创建或更新 `.papercraft/project_memory.json`
- 大项目必须以 main.tex 或当前 TeX 为入口抽样建立项目记忆，不得因为项目全文过长直接中断
- 抽样只纳入 main.tex 及其 `\input` / `\include` 引用的 `.tex/.latex` 正文文件
- 构建项目前必须本地清洗 LaTeX，只保留标题、摘要、章节标题、caption 和自然语言正文段落
- `.sty/.cls/.bst/.rtx`、未被 main 引用的旧 `.tex`、完整 BibTeX、格式命令、公式、表格和代码环境默认不进入项目记忆请求
- 抽样记忆必须标记 sampled，并在 UI 中提示用户确认分类
- 项目记忆必须区分当前论文、模板结构、旧文残留、不确定内容和忽略内容
- 记忆条目包含文件路径、章节或块标题、摘要、关键词、术语、引用线索、来源类型、置信度和判断依据
- 来源类型固定为 current、template、legacy、ambiguous、ignored
- 疑似上一篇文章残留、旧实验结果、旧研究对象、模板占位和示例 BibTeX 不得作为当前论文事实
- 前端提供项目记忆面板，用户可把条目标记为当前论文、模板结构、旧文残留、不确定或忽略
- 用户确认当前分类后，项目记忆面板自动折叠到最小高度，只保留标题、状态和展开入口；已确认记忆遇到普通文本编辑时只更新为“内容已变化”状态，不强制展开；未确认、分析中、分析失败、手动刷新或加载时发现指纹不一致才默认展开
- 用户人工标记写回项目记忆文件，下次加载优先使用人工标记
- 润色和中文生成默认只把高置信 current 记忆作为事实上下文
- template 记忆只能作为结构或写法参考，不得作为研究事实、结果、贡献或结论
- legacy 和 ignored 不发送给润色或中文生成
- ambiguous 默认不作为当前论文事实
- 项目记忆生成或检索失败时，现有润色、中文生成、反馈、PDF 和对话流程必须继续可用

### 5.8 可编辑写作提示词

- AI 控制区提供多行写作提示词编辑框
- 全部版本、保守修订、学术强化、精简表达各自拥有独立默认提示词
- 通过下拉列表切换写作目标时自动显示该目标对应的提示词
- 用户可以自由编辑当前提示词，最多 4,000 个字符
- 用户编辑后的提示词立即保存到浏览器本地，并在下次启动时恢复
- 不同写作目标的自定义提示词必须分别保存，互不覆盖
- 润色请求必须携带当前编辑框的完整内容
- 用户编辑后的提示词是当前写作目标的最终风格要求，优先于系统内置风格描述
- 用户提示词不能覆盖 LaTeX 结构保护、JSON 返回格式和其他安全约束
- 提供“恢复默认”操作，且只重置当前写作目标

### 5.9 上下文系统

- 自动读取选区前 3 个段落
- 自动读取选区后 3 个段落
- 上下文仅用于辅助模型理解
- AI 只能返回选中部分的改写结果

### 5.10 LaTeX 与术语保护

模型提示必须保护：

- LaTeX 命令和环境
- 行内公式和块级公式
- `\cite{}`、`\ref{}`、`\label{}` 等引用结构
- CFD、LES、RANS 等技术缩写
- 用户原有变量名和符号

### 5.11 结果与替换

- 同时显示 A、B、C 三个结果
- 每个结果支持复制
- 每个结果支持一键替换原选区
- 每个结果可显示修改理由，但修改理由不得进入复制和替换文本
- 替换前检查原选区是否发生变化
- 替换操作进入浏览器撤销记录
- 编辑区产生有效选区后，将选区起止位置和原文锁定为论文选区状态
- 点击写作目标、润色按钮、结果区按钮或其他编辑区外控件不得清除论文选区
- 编辑器失焦时，前端必须通过只读镜像高亮层保持锁定论文选区可见，不得通过抢回编辑器焦点影响右侧控件操作
- AI 请求与一键替换必须使用锁定的论文选区，不依赖按钮点击后的浏览器焦点
- 只有用户再次点击、移动光标、输入或重新选择编辑区内容时，才更新或取消原论文选区
- 使用锁定选区前必须检查对应原文是否发生变化

### 5.12 模型接口

- 后端提供 `POST /api/rewrite`
- 后端提供 POST /api/feedback
- 后端提供 POST /api/draft
- 后端提供 POST /api/chat
- 后端提供 POST /api/project-memory/build
- 后端提供 POST /api/project-memory/retrieve
- 模型密钥只在服务端读取
- 使用独立的 `model_config.json` 管理当前模型、接口地址和请求参数
- 每次润色请求前重新读取配置，修改模型信息后无需重启服务
- 使用 `active_provider` 在多个提供商配置之间切换
- 支持 OpenAI Chat Completions 兼容中转站及额外请求头
- 原生适配 Anthropic Messages API 和 Google Gemini generateContent API
- 内置 OpenAI、DeepSeek、通义千问、Kimi、智谱 GLM、硅基流动配置模板
- 支持全局超时、温度和最大输出 Token 配置
- API Key 可以直接配置，也可以使用 `env:变量名` 从环境变量读取
- 未配置当前提供商的 API Key 时使用离线演示模式
- `model_config.json` 必须加入 `.gitignore`
- 保留旧 `AI_API_KEY`、`AI_API_URL`、`AI_MODEL` 环境变量作为配置文件缺失时的兼容回退
- macOS 模型请求优先使用系统 `curl`，避免旧版 LibreSSL 与中转站 TLS/WAF 不兼容
- `curl` 请求不得在命令行参数中暴露 API Key
- 临时请求头文件权限必须为 `600`，并在请求结束后立即删除
- 系统无 `curl` 时回退到 Python urllib

中转站配置字段：

- `type`：使用 `openai_compatible`
- `api_key`：中转站密钥或 `env:变量名`
- `base_url`：服务根地址或完整 Chat Completions 地址
- `model`：中转站支持的模型名称
- `json_mode`：接口是否支持 JSON Mode
- `extra_headers`：中转站要求的额外请求头

### 5.13 逐词 Diff

- 每个 AI 结果显示原文与改写结果的逐词差异
- 删除内容使用红色删除标记，新增内容使用绿色新增标记
- Diff 保留空格、标点和 LaTeX 命令边界
- 大选区超过比较复杂度阈值时使用前后缀降级算法，避免阻塞界面
- Diff 只用于审核，不修改一键替换的最终文本

### 5.14 项目 ZIP 导出

- 支持把当前完整项目导出为 ZIP
- 导出使用当前编辑后的文本内容
- 保留图片、PDF、字体及未知二进制资源
- 生成的 PDF 可以随项目一同导出
- 服务端仅在内存中组装压缩包，不覆盖原项目
- 限制最多 500 个文件、解码后总量 100 MB

### 5.15 LaTeX 项目跳转

- 双击或按住 `Command/Ctrl` 点击 LaTeX 命令触发跳转
- `\cite{}`、`\citep{}`、`\citet{}` 跳转到 BibTeX 条目
- `\ref{}`、`\eqref{}`、`\autoref{}`、`\pageref{}` 跳转到 `\label{}`
- `\input{}`、`\include{}` 跳转到子 TeX 文件
- `\includegraphics{}` 打开对应图片或 PDF 资源
- 找不到目标时显示明确提示，不改变当前文件

### 5.16 搜索替换

- `Command/Ctrl+F` 打开当前文件搜索替换栏
- 输入搜索词时只更新匹配数量和编辑区搜索高亮，不得抢走搜索框焦点或移动编辑器光标
- 在搜索框按 Enter 跳到下一处，Shift+Enter 跳到上一处
- 支持上一处、下一处、替换当前和全部替换
- 显示当前匹配序号和总数
- 关闭搜索栏时必须清空搜索高亮
- 替换结果进入项目自动保存流程

### 5.17 本地 LaTeX 编译

- 后端自动检测 `latexmk`、`xelatex` 或 `pdflatex`
- 项目仅复制到本机临时目录编译，不上传外部服务
- 禁止 shell-escape，完整构建流程最长 90 秒
- 优先使用 `latexmk` 处理多轮编译和参考文献
- 直接使用 XeLaTeX 或 PDFLaTeX 时，检测参考文献并自动执行 BibTeX，随后再排版两次
- 编译错误解析为文件、行号和消息，点击后跳回源码
- 编译成功后生成 PDF，并把 SyncTeX 产物保留在响应中
- PDF 通过本地临时 HTTP 地址提供，支持 HEAD 与 Range 分段读取；前端加载前验证地址可用，失效时回退到本次编译生成的本地 Blob 缓存
- 通过“打开项目”导入的 PDF、图片等二进制资源在编译或导出前优先从文件句柄重新读取；保存项目状态时必须把二进制资源真实内容持久化到 IndexedDB，文件句柄不可用时回退到该资源缓存，避免每次重启后要求重新打开项目
- 二进制资源读取失败且没有缓存可用时必须显示中文可操作提示，说明需要重新打开项目文件夹，不得直接展示 `The object can not be found here`
- 未安装工具链时返回 `toolchain_missing`，不得显示虚假成功状态
- 检查系统 PATH、`/Library/TeX/texbin`、Homebrew、本项目 `tools` 目录和 `LATEX_ENGINE`
- 旧服务返回 404 或非 JSON 内容时，前端必须提示重启服务，不得显示底层对象错误

### 5.18 PDF 预览

- 右侧结果区提供 AI、PDF 与对话三个标签
- 手动编译成功后直接切换到 PDF 标签并显示结果，不弹出编译诊断面板
- 编译失败、工具链缺失或存在需要展示的诊断时，使用编译诊断面板显示错误并支持源码定位
- 支持保存后延迟自动编译并刷新 PDF，自动编译开关常显在编译按钮上方并与编译按钮组成右对齐两行操作组
- PDF 预览地址会加时间戳避免旧缓存；本地 `/api/pdf/` token 失效时自动切换到本次编译响应里的本地缓存，不显示对象不存在页面
- PDF 预览区提供“下载 PDF”和“全屏浏览”按钮；下载使用本次编译生成的 Blob 缓存，全屏浏览直接在新标签打开浏览器独立 PDF 预览器，并请求 250% 缩放，避免 CSS 放大造成清晰度下降
- 手动编译先切换到可见 PDF 标签再加载 iframe；后台自动编译若 PDF 标签未打开，则暂存最新预览地址，等用户打开 PDF 标签后再加载，避免隐藏 iframe 触发浏览器 PDF 对象错误
- 浏览器 PDF 预览加载异常不得覆盖编译成功状态；前端捕获 iframe 加载异常并显示中文提示，避免把底层英文 DOM 错误作为编译失败提示
- 自动编译默认关闭，缺少工具链时自动关闭
- 当前只完成 PDF 实时刷新；基于 SyncTeX 的源码与 PDF 双向定位尚未完成

- 双击启动器后能自动打开应用

- “打开文件”只打开单个可编辑文件，不直接导入 ZIP

- 能从 ZIP 创建可见项目文件夹，重名时自动创建编号目录且不覆盖已有项目
- 能打开 `.docx` 并以纯文本载入正文内容
- 选择旧版 `.doc` 时显示暂不支持提示
- 点击 `中 / EN` 后，主要界面按钮、标题、标签、占位符、状态、空态、PDF 和聊天区文案能在中文与英文之间切换
- 刷新应用后保留上次选择的界面语言，且不改变论文内容、项目文件名、AI 输出、编译日志或用户提示词
- 文件树能显示并打开支持的文件
- `main.tex` 存在时能被优先打开
- 选中文本后能获得三个润色版本，并在模型支持时显示每个版本的修改理由
- 替换不会修改选区之外的内容
- 点击“分析当前文件”后能显示 10 条写作反馈，且不会自动修改源码
- 每条写作反馈都能展开/收起解决方案或推荐文字，并可通过叉号删除
- 每条写作反馈都能跳转到左侧对应原文片段；原文位置变化时提示重新分析
- 输入中文草稿后能生成符合当前写作目标和论文上下文的英文内容
- 点击右侧“中文生成”标签后能显示中文草稿输入面板
- 当前目标为“全部版本”时，中文生成结果显示 A/B/C 三个可编辑候选框，每个候选都能独立复制和插入光标处
- 当前目标为保守、学术或精简时，中文生成结果显示一个可编辑候选框
- 中文草稿生成结果不会自动写入源码，点击对应候选的“插入光标处”后才插入编辑器并可撤销
- 打开项目后能后台生成项目记忆，并在控制区显示分析中、待确认、已就绪或失败状态
- 大型项目能抽样生成项目记忆，不得提示“项目文本过长”
- 用户能把项目记忆条目标记为当前论文、模板结构、旧文残留、不确定或忽略
- 标记为旧文残留或忽略的内容不得进入润色和中文生成请求
- 标记为模板结构的内容只能作为结构参考，不得作为当前论文事实
- 右侧“对话”标签能围绕项目上下文回答问题
- 项目级对话生成的 changes 必须先展示预览，不得自动写入文件
- 点击应用建议修改前必须检查目标文件仍包含原 find 文本
- 未配置模型密钥时仍能完成演示流程
- 模型密钥不会发送到浏览器端
- 打开长篇 `.tex` 文件后，正文和行号能同步、连续地上下滚动
- 关闭并重新启动应用后，能自动恢复上次项目、当前文件和项目资源
- 修改任意可编辑文本后，无需手动操作即可保存到项目缓存
- 通过目录写权限打开的项目，修改后能自动写回对应原文件
- 从 ZIP 创建的项目修改后能自动写回新建项目文件夹
- 无原文件写权限时，状态栏只能显示本地缓存已保存
- 首次打开文本文件时显示文件开头，不得自动滚动到文末
- 切换文件或重启应用后，恢复该文件上次的滚动位置、光标和选区
- 单段 LaTeX 内容很长时，在编辑区内自动软换行且不产生横向滚动
- 拖动区域分隔条时，正文软换行实时响应且源文本不发生变化
- 编辑区、AI 控制区和项目文件树分隔条均可拖动，拖动时即使鼠标移出窄分隔条也不得中断
- 软换行后，逻辑行号与对应文本首行保持对齐
- 重启应用后，各区域恢复用户上次调整的宽度
- 选中文本后切换四个写作目标，编辑区选区保持不变
- 点击润色及结果区按钮时，AI 请求和替换仍使用原选区
- 重新点击或移动编辑区光标后，旧选区才被取消或更新
- 修改 `model_config.json` 的当前提供商后，无需重启即可用于下一次请求
- OpenAI 兼容中转站能通过自定义地址、模型、密钥和额外请求头调用
- Claude 和 Gemini 配置分别生成符合其原生协议的认证头与请求体
